# Maintenance guide

## Weekly

- Triage new issues and pull requests.
- Review dependency and security alerts.
- Confirm `main` is green on Windows, Linux, and macOS.
- Keep `good first issue` and `help wanted` work actionable.

## Monthly

- Run the extended simulation campaign.
- Review platform-support claims and dependency lifecycles.
- Archive stale design proposals with a clear reason.
- Check that contributor documentation still matches the build.

## Quarterly

- Publish a roadmap update.
- Review maintainer coverage and subsystem ownership.
- Test release packages on representative operating systems.
- Review threat model, branch protection, and release permissions.

Never merge around a failing required check merely to publish faster. Document exceptional decisions in the pull request and follow up immediately.
