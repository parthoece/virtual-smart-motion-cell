# Suggested good first issues

These tasks are intentionally small enough for a new contributor to complete without understanding the entire runtime.

1. Add a simulated stack-light adapter and unit specification.
2. Add a browser toggle for commanded-position ghost visibility.
3. Add JSON export for active alarms.
4. Add a new approved example recipe with schema validation.
5. Improve HMI connection-state wording and reconnect feedback.
6. Add a localization key for one HMI card.
7. Add a CSV event exporter behind an application port.
8. Add a viewer keyboard shortcut and document it.
9. Add a specification for an unknown or empty command type.
10. Add a Linux ARM64 contributor setup note.

Each issue should state expected files, acceptance criteria, tests, compatibility impact, and whether a design proposal is required. Maintainers should convert these suggestions into tracked issues after publication.
