# Governance

The project uses a maintainer-led, consensus-seeking model.

- Contributors propose changes through issues, discussions, and pull requests.
- Maintainers review quality, architecture, security, compatibility, and maintenance cost.
- Significant changes require a documented proposal and ADR.
- Two maintainer approvals are preferred for security-sensitive or breaking changes once the project has multiple active maintainers.
- Maintainers may become inactive after six months without project activity and can return through normal contribution.
- Governance disputes should be documented and resolved transparently when privacy and security allow.
